# Governance

Aziky currently uses a maintainer-led governance model.

Maintainers are responsible for the language contract, compiler architecture,
release quality, repository access, security response, community enforcement,
and project roadmap. Routine decisions are made through issue and pull-request
discussion. Maintainers seek technical consensus, but may make a final decision
when consensus is not available.

Changes to syntax, ownership, ABI, package formats, artifact compatibility,
supported targets, licensing, or governance require an issue with a written
rationale before implementation. Accepted decisions must be reflected in the
relevant reference documentation and tests.

Regular contributors may be invited to become maintainers based on sustained
high-quality contributions, sound technical judgment, respectful collaboration,
and willingness to perform review and maintenance work. Maintainer status may
be relinquished voluntarily or removed after prolonged inactivity, serious
security/process failures, or Code of Conduct violations.

The project may adopt a more formal proposal and voting process as the number of
maintainers and downstream users grows.
